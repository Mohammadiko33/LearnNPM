1. package.json جی هست ؟
    یک فایلی هست که با فرمت و سینتکس جیسون هسا=ت محتویاتش و میاد برای من اطلاعات و حزعیات پروژ که العان دارم باش کار میکنمرو داخل خودش ذخیره میکنه 
    چند تا مثال در مورد پکیح جیسون
    اسم پروژم چیه / العان دارم کدوم ورژن از پروژمو توسعه میدم / نویسنده این پروژه کیه / تحت چه دسترسی و لیسانیسی هستش

2.  package name : اسم پروژه
    version : نسخه پروژه که از سه بخش تشکیل شده و بخش هارو جلوتر توضیح میدم
    description : توضیحات پروژه
    entry point : یعنی نقطه شروع یعنی نقطه استارت یعنی اگر شخصی خاست این پکیج و پروژه شما رو استفاده بکنه وقتی اینپورتش کرد کدوم فایل از پروژه برای اون شخص اجرا بشه یعنی کدام فایل شما نقشه شروع پروژه شماس
    test command : پروژه های ما کلا توی سه فاز نوشته میشن فار اول فازدلپمنت هست فاز بعدی فاز تست هست فاز اخر فاز پروداکشن هست 
        توی فاز دولوپمنت ما برنامه نویس ها میام پروژه خودمون رو کد نویسی و توسعه شون میدیم وقتی که فاز پروداکشن تموم شد پروژه ما میره روی فاز دوم 
        فاز دوم که تستر های پروژه تستر های تیم میان برای پروژه ما تست مینویسن و اگر طی این تست پروژمون باگی نداشته باشه پروژه میره روی فاز پروداکشن و دیگه دپلوی میشه روی هاست اپلود میشه تحویل مشتری داده میشه اما اگر توی مرحله تست پروژه مشکلی داشته باشه و ازش مشکلی نشون داده بشه و و مشکلی ازش بفهمیم و خارح بکنیم این پروژه میره به فاز دیسکریپشن و یا همون دولوپمنت و دوباره توسعه داده میشه و اون باگ های فیکس میشن و دوباره اون تست میشه و میره توی فاز پروداکشن توی این کامند میگه که خاست که کامند 
    npm test رو بزنه چه اتفاقی بیوفته
    git repository: کل ریسورس های پروژمون رو روی یک ریپوزیتوری قرار میدیم توی این قسمت ریپوزیتوری گیت ازمون میخاداما ما چون اطلاعاتی از گیت نزاشتیم از این پروژمون توی این قسمتم چیزی رو وارد نمیکنیم
    keywords : دقیقا مثل تگ اینستا و یوتوب عمل میکنه
    author : اسم نویسنده 
    license : لیسانیس های خیلی زیادی رو داریم ما مثل همین مثال های زیر و چیزای دیگه که این لاینسس میاد برای ما تعیین میکنه این پکیج ما وقتی میخام یک نفر از این پکیج ما استفاده بکنه تحت چه شرایط و قوانین میتونه استفاده بکنه
